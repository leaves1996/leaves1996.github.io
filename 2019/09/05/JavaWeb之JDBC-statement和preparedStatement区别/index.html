本文小编就说说jdbc中statement和preparedStatement的区别。

1.性能区别

  执行的时候:

  由上可以看出，PreparedStatement有预编译的过程，已经绑定sql，之后无论执行多少遍，都不会再去进行编译，而 statement 不同，如果执行多少遍，则相应的就要编译多少遍sql，所以从这点看，preStatement 的效率会比 Statement 要高一些

4.继承关系

  • 作为 Statement 的子类，PreparedStatement 继承了 Statement 的所有功能，

5、批处理时的区别：

  • Statement：批处理可以处理不同的sql语句，静态sql
  • preparedStatement：批处理只能处理同一个sql语句，动态sql。
